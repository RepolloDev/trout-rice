<h1 align="center">Problema 🤯</h1>

Realizar el diseño de base de datos de una biblioteca universitaria, con opción de extender el modelo y su complejidad. Debe tener lo siguiente:

1. **Requerimientos**: el análisis de los requerimientos
2. **Diseño conceptual**: el modelo entidad relación.
3. **Diccionario de datos**: el conjunto de elementos dentro del modelo.
4. **Diseño lógico**: el modelo relacional.
5. **Diseño físico**: el esquema de la base de datos.

## 📚 Biblioteca

En una biblioteca universitaria se maneja información sobre `autores`, `libros`, `ejemplares`, `usuarios` y `prestamos`. Además, tenemos:

- Un autor **tiene** varios libros escritos.
- Un libro **es escrito** por uno o varios autores.
- Un libro **tiene** varios ejemplares.
- Un usuario puede **prestarse** uno o varios ejemplares.

> [!NOTE]
> En este escenario un usuario es un estudiante de la carrera.

> [!TIP]
> Los ejemplares se refieren a las copias físicas del libro, pero esto a nivel biblioteca, no a nivel mundial, no tendría sentido guardar eso.

> [!TIP]
> Un usuario puede prestarse dos o más ejemplares de un libro, pero no debería poder prestarse dos o más ejemplares de un mismo libro, porque eso no tiene sentido.
