<h1 align="center">Definiciones 🎯</h1>

## 🎈 Contexto

- **Institución**: una universidad.
- **Dominio**: la biblioteca de una carrera específica de la universidad.
- **Usuarios finales**: estudiantes de la carrera y personal de la biblioteca.
- **Materiales**: colección física de ejemplares de libros.

## 🎯 Objetivo del documento

Definir el alcance, las reglas de negocio y los requerimientos del sistema de biblioteca con un enfoque **simple y básico**, manteniendo la lógica en la aplicación y sin usar objetos almacenados en base de datos.

## 🚧 Alcance

- Gestión de autores.
- Gestión de editoriales.
- Gestión de categorías.
- Gestión de libros.
- Gestión de ejemplares físicos de la biblioteca.
- Gestión completa de estudiantes en la base de datos del sistema.
- Registro de préstamos y devoluciones.
- Registro manual de incidencias.
- Consulta de disponibilidad de ejemplares.

## 🪁 Fuera de alcance

## 🧱 Modelo operativo del sistema

### Estado del ejemplar (operativo y borrado lógico)

| Estado | Significado |
| --- | --- |
| `activo` | El ejemplar está en circulación. La disponibilidad final se calcula con préstamo y estado físico. |
| `retirado` | El ejemplar queda fuera de circulación permanente (también actúa como borrado lógico del ejemplar). |

### Estado físico del ejemplar

| Estado | Significado | Observaciones |
| --- | --- | --- |
| `nuevo` | Condición perfecta, sin uso notable. | Libre para prestar. |
| `bueno` | Desgaste normal de uso. | Libre para prestar. |
| `dañado` | Con defectos que comprometen su uso normal. | **Bloquea préstamo**. |

### Estado del estudiante

| Estado | Significado |
| --- | --- |
| `activo` | Puede usar el servicio de préstamo. |
| `suspendido` | No puede realizar préstamos. |

### Estado del préstamo (calculado)

- Si `fecha_devolucion` es `NULL`, el préstamo está **abierto**.
- Si `fecha_devolucion` tiene valor, el préstamo está **cerrado**.

### Disponibilidad del ejemplar (calculada)

Un ejemplar está disponible si cumple todas estas condiciones:

- `estado = activo`
- `estado_fisico != dañado`
- No existe préstamo abierto del ejemplar (`fecha_devolucion IS NULL`)

## 💼 Reglas de negocio

- **Retiro de circulación**: un ejemplar marcado como `retirado` no puede ser prestado bajo ningún concepto.
- **Prestar ejemplares**: solo si el ejemplar está disponible según cálculo.
- **Registro de préstamo**: cada préstamo registra estudiante, ejemplar, fecha de préstamo y fecha de vencimiento.
- **Cálculo de vencimiento**: fecha de préstamo + 14 días calendario.
- **Máximo de préstamos activos**: un estudiante puede tener como máximo 4 préstamos abiertos.
- **Un ejemplar por título**: un estudiante no puede tener más de un préstamo abierto del mismo título.
- **Registro de devolución**: se registra la fecha de devolución del ejemplar.
- **Sin renovaciones**: los préstamos no se renuevan; al vencimiento solo corresponde devolución.
- **Sin reservas**: no existe cola de espera ni apartados de ejemplares.
- **Incidencias manuales**: el bibliotecario registra incidencias asociadas a préstamo y estudiante, con fecha y descripción.
- **Histórico permanente**: las incidencias no se eliminan del historial.
- **Suspensión administrativa**: el bibliotecario puede suspender o reactivar estudiantes.

> [!NOTE]
> Ejemplos de incidencia: devolución fuera de plazo, devolución con daño, no devolución del ejemplar.

## 📅 Políticas operativas

### Plazos

| Concepto | Regla |
| --- | --- |
| Duración estándar de préstamo | 14 días calendario |
| Cálculo de vencimiento | Fecha de préstamo + 14 días |
| Retraso | Todo retraso se considera incumplimiento y puede registrarse como incidencia |

### Cupos

| Concepto | Límite |
| --- | --- |
| Máximo de préstamos abiertos por estudiante | 4 |
| Máximo de ejemplares del mismo título por estudiante (simultáneo) | 1 |

## ✍️ Casos de uso

### Actores

- **Bibliotecario**: operador principal del sistema.
- **Estudiante**: usuario del servicio de préstamos (actor de dominio, sin operación directa en este alcance).

### Casos de uso principales

| ID | Caso de uso | Actor principal | Resultado esperado |
| --- | --- | --- | --- |
| CU-01 | Gestionar catálogo | Bibliotecario | Registros de autores, editoriales, categorías y libros actualizados. |
| CU-02 | Registrar ejemplar | Bibliotecario | Ejemplar creado con estado operativo y estado físico. |
| CU-03 | Registrar préstamo | Bibliotecario | Préstamo abierto con fecha de vencimiento. |
| CU-04 | Registrar devolución | Bibliotecario | Préstamo cerrado con fecha de devolución. |
| CU-05 | Registrar incidencia | Bibliotecario | Incidencia asociada a préstamo y estudiante persistida en historial. |
| CU-06 | Suspender/reactivar estudiante | Bibliotecario | Estado del estudiante actualizado. |
| CU-07 | Consultar disponibilidad | Bibliotecario | Lista de ejemplares disponibles por libro. |
| CU-08 | Consultar historial | Bibliotecario | Historial de préstamos e incidencias por estudiante o ejemplar. |

### Flujos principales

1. **Flujo de préstamo**
	- Bibliotecario selecciona estudiante y ejemplar.
	- El sistema valida reglas (estado del estudiante, cupos, disponibilidad, préstamo abierto y límite por título).
	- El sistema registra préstamo y fecha de vencimiento.
	- El sistema confirma la operación.

2. **Flujo de devolución**
	- Bibliotecario identifica el préstamo abierto.
	- El sistema registra fecha de devolución.
	- El préstamo se considera cerrado por tener `fecha_devolucion`.
	- Si corresponde, bibliotecario registra incidencia.

3. **Flujo de retiro de ejemplar**
	- Bibliotecario selecciona ejemplar.
	- El sistema cambia estado operativo a `retirado`.
	- El ejemplar queda fuera de circulación.
